# Requirements: Google ADK Workshop - Context-Engineered Travel Booking Assistant

**Defined:** 2026-01-23
**Core Value:** Participants learn to build production-ready AI agents with proper context engineering - combining real-time APIs, knowledge bases, and structured data

## v1 Requirements

Requirements for the 90-minute workshop. Each maps to roadmap phases.

### Workshop Materials

- [x] **WORK-01**: Participant can follow setup guide to configure Python 3.11/3.12, ADK 1.23.0, and GCP credentials
- [x] **WORK-02**: Participant can complete Exercise 1 (basic agent with Gemini 2.5 Flash)
- [x] **WORK-03**: Participant can complete Exercise 2 (function calling with flight/hotel search)
- [x] **WORK-04**: Participant can complete Exercise 3 (RAG integration with destination knowledge)
- [x] **WORK-05**: Participant can complete Exercise 4 (session management for preferences)
- [x] **WORK-06**: Participant can reference complete solutions for all exercises
- [x] **WORK-07**: Participant can use troubleshooting guide to resolve common errors
- [x] **WORK-08**: Participant can follow deployment guide to publish agent to Vertex AI
- [x] **WORK-09**: Participant can run reference implementation (complete working agent)
- [x] **WORK-10**: Participant can understand context engineering module (tools vs RAG vs sessions decision framework)
- [x] **WORK-11**: Participant can use production readiness checklist to evaluate agent quality
- [x] **WORK-12**: Pre-workshop validation system verifies participant environment 48h ahead
- [x] **WORK-13**: Participant can use git branch checkpoints to jump to any exercise

### Agent Capabilities

- [x] **AGENT-01**: Agent can search flights by destination, dates, and number of passengers
- [x] **AGENT-02**: Agent can search hotels by location, check-in/out dates, and number of guests
- [x] **AGENT-03**: Agent can retrieve destination information from knowledge base (visa requirements, attractions, weather)
- [x] **AGENT-04**: Agent can remember user preferences across conversation (budget, travel style, dietary restrictions)
- [x] **AGENT-05**: Agent can maintain multi-turn conversations with context retention
- [x] **AGENT-06**: Agent can handle errors gracefully and provide helpful error messages
- [x] **AGENT-07**: Agent can provide smart recommendations combining real-time availability with destination knowledge
- [x] **AGENT-08**: Agent can filter results by user's stated budget
- [x] **AGENT-09**: Agent can infer unstated preferences from conversation context

### Technical Infrastructure

- [x] **INFRA-01**: Mock flight API provides search results without rate limits or costs
- [x] **INFRA-02**: Mock hotel API provides search results without rate limits or costs
- [x] **INFRA-03**: Destination knowledge corpus with 10-15 PDF guides is pre-indexed in Vertex AI RAG
- [x] **INFRA-04**: Pre-provisioned GCP accounts have Vertex AI API enabled
- [x] **INFRA-05**: Workshop materials run in Google Colab for zero-install experience
- [x] **INFRA-06**: pytest test framework with ADK AgentEvaluator validates agent behavior
- [x] **INFRA-07**: Agent deploys to Vertex AI Agent Engine with shareable endpoint
- [x] **INFRA-08**: Cost monitoring dashboard shows token usage and API costs

### Context Engineering

- [x] **CONTEXT-01**: Workshop explains when to use function calling vs RAG vs session state
- [x] **CONTEXT-02**: Workshop demonstrates real-time data access pattern (function calling for flights/hotels)
- [x] **CONTEXT-03**: Workshop demonstrates static knowledge retrieval pattern (RAG for destination guides)
- [x] **CONTEXT-04**: Workshop demonstrates state management pattern (sessions for user preferences)
- [x] **CONTEXT-05**: Workshop demonstrates prompt engineering for effective agent instructions
- [x] **CONTEXT-06**: Workshop demonstrates error handling patterns (errors in context improve behavior)
- [x] **CONTEXT-07**: Workshop demonstrates cache-friendly patterns (avoid cache-breaking changes)

## Out of Scope

Explicitly excluded to keep workshop focused and within 90-minute timeline.

| Feature | Reason |
|---------|--------|
| Custom model fine-tuning | Beyond ADK fundamentals, requires ML expertise not in scope |
| Payment processing | Focus on agent capabilities, not business logic integration |
| Real booking system integration | Legal/compliance complexity, use mock data for teaching |
| Advanced RAG techniques | Embedding tuning, hybrid search too advanced for 90-minute beginner workshop |
| Multi-language support | English-only keeps scope manageable |
| Mobile app interface | Web-based Colab and API endpoints sufficient for teaching |
| Production monitoring/observability | Basic deployment only, full monitoring deferred to advanced topics |
