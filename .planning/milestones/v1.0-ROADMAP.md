**Status:** SHIPPED 2026-01-24

## Overview

Complete 90-minute hands-on workshop teaching Google Agent Development Kit (ADK) through building a travel booking assistant. Demonstrates context engineering by combining real-time APIs, RAG knowledge retrieval, and session state management.

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Colab-first approach | Eliminates local environment setup for 90% of participants | ✓ Good |
| Mock APIs over real travel APIs | No costs, rate limits, or API key management | ✓ Good |
| Error-in-context pattern | LLM sees errors and can help users fix requests | ✓ Good |
| Google AI API key (not Vertex AI) | Simpler participant experience | ✓ Good |
| Hybrid agent for RAG constraint | ADK can't mix RAG with function tools | ✓ Good |
| State prefixes (user:, temp:, app:) | Clear persistence scope semantics | ✓ Good |
| Three-layer solution format | Explains WHY patterns work, not just WHAT | ✓ Good |
| 48-hour pre-validation | Catches setup issues before workshop day | ✓ Good |

**Issues Resolved:**

- ADK VertexAiRagRetrieval single-tool constraint → Hybrid agent pattern
- Colab nested event loop → Top-level await instead of asyncio.run()
- ToolContext parameter naming → Must be `tool_context` for framework injection

**Issues Deferred:**

- Real travel API integration (kept mock for simplicity)
- Multi-agent orchestration (too advanced for 90-minute workshop)
- Production monitoring/observability (basic deployment only)

**Technical Debt:**

- tests/test_travel_agent.py has placeholder tests for error cases and multi-turn flow (INFO severity)
