Create a shell script that creates checkpoint branches from main.

Script requirements:
1. Creates 4 checkpoint branches:
   - checkpoint/exercise-1: After Exercise 1 complete (basic agent working)
   - checkpoint/exercise-2: After Exercise 2 complete (function calling working)
   - checkpoint/exercise-3: After Exercise 3 complete (RAG integration working)
   - checkpoint/exercise-4: After Exercise 4 complete (state management working)

2. Script structure:
```bash
#!/bin/bash
# create-checkpoints.sh - Create workshop checkpoint branches
# Run this from the repository root after completing all exercises

set -e  # Exit on error

echo "Creating workshop checkpoint branches..."
echo "========================================"

# Ensure we're on main with latest code
git checkout main
git pull origin main 2>/dev/null || true

# Function to create checkpoint branch
create_checkpoint() {
    local branch_name=$1
    local description=$2

    echo ""
    echo "Creating $branch_name..."

    # Delete local branch if exists
    git branch -D "$branch_name" 2>/dev/null || true

    # Create new branch from main
    git checkout -b "$branch_name"

    # Push to remote (if remote exists)
    git push -u origin "$branch_name" --force 2>/dev/null || echo "  (Skipped push - no remote)"

    # Return to main
    git checkout main

    echo "  ✓ $branch_name created - $description"
}

# Create all checkpoints
# Note: All checkpoints point to main (complete implementation)
# The documentation explains which files to focus on for each exercise

create_checkpoint "checkpoint/exercise-1" "Basic agent working"
create_checkpoint "checkpoint/exercise-2" "Function calling tools working"
create_checkpoint "checkpoint/exercise-3" "RAG integration working"
create_checkpoint "checkpoint/exercise-4" "State management working (complete)"

echo ""
echo "========================================"
echo "✓ All checkpoint branches created!"
echo ""
echo "Participants can catch up with:"
echo "  git checkout checkpoint/exercise-2"
echo ""
echo "To see all checkpoints:"
echo "  git branch -a | grep checkpoint"
```

```markdown
# Workshop Checkpoint Branches

## Overview

If you fall behind during the workshop, you can use checkpoint branches to catch up to the current exercise. Each checkpoint contains the complete working code up to that point.

## Available Checkpoints

| Checkpoint | What's Working | When to Use |
|------------|----------------|-------------|
| `checkpoint/exercise-1` | Basic ADK agent | Start of Exercise 2 |
| `checkpoint/exercise-2` | Function calling tools | Start of Exercise 3 |
| `checkpoint/exercise-3` | RAG integration | Start of Exercise 4 |
| `checkpoint/exercise-4` | State management | Complete implementation |

## How to Catch Up

### Step 1: Save your current work (optional)

If you want to save your progress:
\`\`\`bash
git stash
\`\`\`

### Step 2: Switch to checkpoint

Replace `exercise-2` with the checkpoint you need:
\`\`\`bash
git checkout checkpoint/exercise-2
\`\`\`

### Step 3: Continue with the workshop

You now have working code from previous exercises. Continue with the current exercise notebook.

### Step 4: Restore your work later (optional)

After the workshop, if you want your original work back:
\`\`\`bash
git checkout main
git stash pop
\`\`\`

## What Each Checkpoint Contains

### checkpoint/exercise-1
- `01-hello-agent.ipynb`: Basic agent complete
- `reference-implementation/agent.py`: Basic agent definition

### checkpoint/exercise-2
- Everything from exercise-1, plus:
- `02-tools-functions.ipynb`: Function calling complete
- `reference-implementation/tools.py`: search_flights, search_hotels

### checkpoint/exercise-3
- Everything from exercise-2, plus:
- `03-rag-knowledge.ipynb`: RAG integration complete
- `reference-implementation/rag_tools.py`: Destination knowledge retrieval
- `reference-implementation/hybrid_agent.py`: Hybrid coordinator pattern

### checkpoint/exercise-4
- Complete workshop implementation
- `04-sessions-state.ipynb`: State management complete
- `reference-implementation/state_utils.py`: Preference management

## Troubleshooting

### "Your local changes would be overwritten"

Your uncommitted changes conflict with the checkpoint. Options:
1. Save your work: `git stash`
2. Discard your work: `git checkout .` (warning: loses changes)

### "branch already exists"

Delete local branch first:
\`\`\`bash
git branch -D checkpoint/exercise-2
git checkout checkpoint/exercise-2
\`\`\`

### Can't find the checkpoint branch

Make sure you have the latest branches:
\`\`\`bash
git fetch origin
git checkout checkpoint/exercise-2
\`\`\`

## For Instructors

Create checkpoint branches before workshop delivery:
\`\`\`bash
./workshop-materials/scripts/create-checkpoints.sh
\`\`\`
```
