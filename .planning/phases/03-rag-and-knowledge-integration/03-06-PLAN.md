**1. Tools vs RAG Decision Framework** (add after tools section)
```markdown
## Context Engineering: Choosing the Right Approach

The travel assistant uses three context patterns. Choose based on data characteristics:

### Decision Flowchart

```
Question Type?
      |
      v
Real-time data needed?  ----YES----> FUNCTION CALLING
(availability, prices)              (search_flights, search_hotels)
      |
      NO
      |
      v
Static knowledge needed? ----YES----> RAG RETRIEVAL
(guides, policies, facts)            (destination_knowledge)
      |
      NO
      |
      v
User preference tracking? ---YES----> SESSION STATE
(budget, style, history)             (Exercise 4)
      |
      NO
      |
      v
LLM answers directly
(general knowledge, reasoning)
```

### Pattern Comparison

| Aspect | Function Calling | RAG Retrieval | Session State |
|--------|-----------------|---------------|---------------|
| **Data Type** | Real-time | Static docs | User context |
| **Example** | Flight prices | Visa rules | Budget pref |
| **Latency** | API call (ms) | Vector search (ms) | In-memory |
| **Update Freq** | Live | Periodic | Per session |
| **ADK Tool** | Python function | VertexAiRagRetrieval | SessionService |

### When to Use Each

**Function Calling (Exercise 2)**
- Booking availability and pricing
- Weather forecasts (live API)
- Currency conversion
- Any data that changes frequently

**RAG Retrieval (Exercise 3)**
- Destination guides and travel tips
- Visa requirements (changes slowly)
- Cultural customs and etiquette
- Company policies and documentation

**Session State (Exercise 4)**
- User budget preferences
- Dietary restrictions
- Travel style (luxury vs budget)
- Conversation history
```

**2. RAG Integration Section**
```markdown
## RAG Knowledge Integration

The agent retrieves destination information from a pre-indexed knowledge corpus.

### Files

| File | Purpose |
|------|---------|
| `rag_tools.py` | RAG tool configuration with explicit descriptions |
| `hybrid_agent.py` | Multi-agent coordination (tools + RAG) |

### Quick Start

```python
from rag_tools import create_destination_knowledge_tool

# Create RAG tool (requires RAG_CORPUS_ID env var)
destination_knowledge = create_destination_knowledge_tool()

# Use in agent
agent = Agent(
    model='gemini-2.5-flash',
    name='destination_expert',
    tools=[destination_knowledge],  # ONLY RAG tool
    instruction='...'
)
```

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `RAG_CORPUS_ID` | Yes (for RAG) | Full corpus path |

Format: `projects/{project}/locations/{location}/ragCorpora/{id}`

### Important Constraint

**VertexAiRagRetrieval cannot be mixed with function tools in the same agent.**

This is an ADK architecture constraint. For hybrid capabilities:

```python
# WRONG - will fail
agent = Agent(
    tools=[search_flights, search_hotels, destination_knowledge]  # ERROR
)

# CORRECT - separate agents
booking_agent = Agent(tools=[search_flights, search_hotels])
destination_agent = Agent(tools=[destination_knowledge])
# Then coordinate (see hybrid_agent.py)
```
```

**3. Hybrid Agent Documentation**
```markdown
## Hybrid Agent Pattern

For comprehensive travel assistance, combine booking tools and RAG knowledge.

### Usage

```python
from hybrid_agent import HybridTravelAssistant
import asyncio

async def main():
    assistant = HybridTravelAssistant()

    # Booking query - routes to booking agent
    response = await assistant.assist(
        "Find flights from SFO to Tokyo on March 15"
    )

    # Knowledge query - routes to destination agent
    response = await assistant.assist(
        "What are the visa requirements for Japan?"
    )

    # Mixed query - booking + enrichment
    response = await assistant.assist(
        "Find hotels in Paris and tell me about the best neighborhoods"
    )

asyncio.run(main())

# Or use convenience function
from hybrid_agent import hybrid_travel_assistant
response = await hybrid_travel_assistant("Your query here")
```

### How Routing Works

1. **Intent Detection**: Keywords classify query type
   - Booking: "flight", "hotel", "book", "price", "availability"
   - Knowledge: "visa", "weather", "culture", "attraction", "tips"

2. **Agent Selection**: Route to appropriate agent

3. **Enrichment**: Booking queries can be enriched with destination tips

### Architecture

```
User Query
     |
     v
[Intent Detection]
     |
     +---> Booking keywords ---> Booking Agent ---> [Optional: Enrich]
     |                                                      |
     +---> Knowledge keywords --> Destination Agent         |
     |                                                      v
     +---> Default -----------> Booking Agent         Combined Response
```
```

**4. RAG Troubleshooting Section**
```markdown
## Troubleshooting

### RAG Issues

| Error | Cause | Solution |
|-------|-------|----------|
| "RAG corpus ID not provided" | Missing env var | Set `RAG_CORPUS_ID` |
| "Tool cannot be combined" | Mixed tools in agent | Use separate agents |
| Poor retrieval quality | Bad chunking | Check corpus config |
| No results returned | Threshold too high | Lower `vector_distance_threshold` |
| Irrelevant results | Threshold too low | Raise threshold to 0.6+ |

### Checking Corpus Status

```bash
# Check if corpus is indexed
gcloud ai rag-corpora describe ${CORPUS_ID} --location=us-central1

# List files in corpus
gcloud ai rag-files list --rag-corpus=${CORPUS_ID} --location=us-central1
```

### Testing Retrieval Quality
